# The Fog Language specification

This book defines the rules for compiling Fog source code files.

The purpose of this book is to define the grammar of the language in a clear and accessible way, as well as to explain some of the less obvious nuances. The book also provides a list of assertions that a compiler must satisfy.

## Special designations
The following designations are found in the book and in the chapters of the book:
- *Italics* are used to highlight the term being defined.
- {scmp}`Small capitals` are used to highlight common words that are used in a special meaning defined in the glossary of the current chapter or the book as a whole; they are also used for abbreviations within the text and for Unicode normative notation.
